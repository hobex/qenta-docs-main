3D Secure 2 - QTILL Checkout Enterprise Update Guide

3D Secure 2 - QTILL Checkout Enterprise Update Guide

Introduction

Qenta Payment CEE is obliged to comply with the 3D Secure 2 standards, which are prescribed in the Payment Services Directive (PSD 2).

For these reasons and to ensure the highest level of security, merchants using our QTILL Checkout Enterprise solution are asked to make certain adjustments. The following adaptations which are described in the upcoming sections below include downloadable updates to the latest versions of LHL and JAPI to process 3D Secure 2.

For more information regarding 3DS2, please visit the 3D Secure 2 page.

Java version

We strongly recommend to use the Java version 1.8.

Downloads and instructions

Please refer to the following sections matching your environment.

LHL/Voyager Version 3.6.x

You have to add additional 3DS2 parameter to your integration. You can find them in the following QTILL-Serial-API 3.23 documentation. The documentation is only available in german.

qtill-serial-api_3.23.doc

JAPI Version 3.9.0 and 4.8.2

You have to update your integration to version 4.8.10 and use the new JAR library. You can download the new version here:

qtill-japi-4.8.10.zip

In order how to use 3D Secure 2, you have to unzip the included javadoc file and go to the index.html file to open the java documentation. After opening the documentation you can search for the 3DS2 use cases. Otherwise you can directly open resources\3ds2_use_cases.html.

warning: for the version 3.9.0 you also have to adapt JAPI methods i.e. initiatePayment, deposit.

Alternative Solutions

Qenta offers multiple payment solutions. You can visit this guide if you want to find a product which best suits your business requirements.
