Order flow diagram

Order flow diagram

Note that mpass has become outdated and is no longer supported by Wirecard as of 2016-09-30.

The following order flow diagram is intended to give an abstract description of the order flow behavior based on mpass transactions.

To interprete this diagram, the successful completion of a NEW PAYMENT is assumed. Starting from this operation it is possible to carry out different transaction-based operations such as deposit, depositReversal, approveReversal, etc. which are described in transaction-based operations for Wirecard Checkout Page and Wirecard Checkout Seamless.

For the explanation of the different status descriptions included in this diagram have a look at payment.{n}.{m}.state, order.{n}.state and credit.{n}.state which are response parameters of the back-end operation getOrderDetails.

Please refer to day-end closings and uncaptured approvals for additional details you need to be aware of.
