HTTP Basic Auth - Merchant Migration Guide

HTTP Basic Auth - Merchant Migration Guide

Introduction

Wirecard is obliged to comply with the latest security standards required by the PCI Data Security Standard (PCI DSS) which involves the secure transmission of sensitive card holder data as well as the use of the latest Java versions.

In this respect and in order to guarantee the highest level of security, merchants using our solution Wirecard Checkout Enterprise (QTill) are requested to carry out certain adaptations. These adaptations which are described in detail in the following step-by-step instructions include new credentials for HTTP Basic Auth and the update process to the latest versions of Java and JAPI.

As a result of Wirecard’s efforts to comply with PCI DSS regulations, we also created a new endpoint for Wirecard Checkout Enterprise which is set in the new configuration.

HTTP Basic Auth for QTill/JAPI and QTill/LHL/Voyager

HTTP Basic Auth and a public server certificate are used in order to meet PCI DSS requirements. For this purpose, a username and a password are required.

As of June 5th, 2018, HTTP Basic Auth is mandatory.

Java version

Due to changed cipher-suites on the new endpoints, only Java version 1.6 or higher is supported.
We strongly recommend to use the current Java version 1.8.

New Endpoints

  • Settings accept only one URL.

  • Within qtill.properties, naming has changed from QTILL_HOST_URL1 and QTILL_HOST_URL2 to QTILL_HOST_URL only.

Please be aware of the fact that we changed the custom ports 8443/9443 to standard https port 443.
Keep firewall settings in mind during update.

Authentication

Authentication has changed from client certificates to username/password.

Old Property New Property Transformation

BASIC_AUTH_USERNAME

File name without path and without extension.

KEYSTOREPASSWORD

BASIC_AUTH_PASSWORD

Password remains the same.

Truststore

The new endpoint is using a server certificate from a different certificate authority.
Therefore, the existing truststore file has to be replaced with the one in the following archive.
As of now, Demo- and production-endpoint have the same Hostname and therefore same CA and in consequence same truststore in place.

Logging (only LHL/voyager)

The logging subsystem has been changed to log4j2 implementation for LHL/voyager. Therefore, a standard log4j2 configuration is required.
The former LOGPATH and LOGROOT configurations within qtill.properties are no longer supported.
