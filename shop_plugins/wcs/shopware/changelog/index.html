Version 1.10.14 (2018-10-02)

  • Implement Installment by RatePAY.

  • Add financialinstitutions for payment methods eps-Überweisung and iDEAL.

Version 1.7.26 (2016-10-12)

  • Compatibility with Shopware version 5.2.8.

  • Check for duplicate requests caused by refreshing or back-button events.

Version 1.7.24 (2016-10-04)

  • Compatibility with Shopware version 5.2.7.

  • Prefix of shop changed to a different variable.

  • Descriptions and configurations deleted only when uninstalling and not on reinstall or update.

  • Configuration possibility for payolution terms checkbox in back-end.

  • Configuration possibility for payolution mID in back-end.

  • Birthday check at Invoice/Installment in front-end visible/changeable.

  • Update of payolution text.

  • Confirmation mails only sent if payment is successful.

  • Information mail for orders changing from pending to failure.

  • Configuration possibility for sending information mails if payment is pending.

  • Update of translations.

  • Removal of check for general terms and conditions with JavaScript for shop version higher than 5.

  • Removal of JavaScript errors.

Tested version: Shopware 5.2.7 with PHP 5.4

Version 1.6.1

  • Improvements regarding orderId and pending state.

Version 1.6.0

  • New fields “SAQ A compliance”, "Iframe CSS URL", “Show Cardholder field”, “Show CVC field” and "Show CreditCard issue date" added.

Version 1.5

Version 1.5.0

  • Update of listed banks which suppport eps Online-Überweisung.

  • Update of listed banks which support iDEAL.

  • New payment method My Voucher added.

  • Improvements regarding the use of the correct currency instead of default currency for saved orders.

Version 1.4

Version 1.4.2

  • Update regarding generation of parameter orderReference to prevent timeouts.

Version 1.4.1

  • New field "Shopware transaction ID" added.

Version 1.4.0

  • Pending message for paymentState PENDING added.

  • CCARD-MOTO and Maestro data storage access improved.

  • Exception if no shopId is given.

  • New payment methods added:

    • ePay.bg

    • TatraPay

  • paymentType MAESTRO no longer supported; Maestro SecureCode now handled via paymentType CCARD.

Version 1.3

Version 1.3.1

  • Plugin updated and improved.

  • For payment method Przelewy24:

    • Special characters removed from parameter orderReference.

    • Sending of consumer e-mail included.

Version 1.3.0

  • Plugin updated and improved.

  • Added paymentType SEPA-DD (formerly ELV) and Trustly (formerly INSTANTBANK).

Version 1.2

Version 1.2.2

  • Plugin updated and adapted regarding optional request parameter orderReference for payment method Bancontact/Mister Cash which allows exactly 10 characters.

Version 1.2.1

  • Plugin updated and adapted regarding optional request parameter orderReference for payment method iDEAL which only allows 16 characters.

Version 1.2.0

  • Added support regarding SEPA (default configuration changed to IBAN/BIC).

  • Plugin updated to Shopware 4.1.4.

Version 1.1

Version 1.1.1

  • Changed URL creation from requests getRoutePath to routers assemble method to support virtual paths.

Version 1.1.0

  • Updates regarding Shopware 4.1

  • Support of additional payment methods:

    • eKonto

    • Instant Bank

    • MPASS

    • Skrill Direct

    • Skrill Digital Wallet

  • Improvements

    • Using Shopware in a sub-directory on your web-server is now supported

    • "AGB checkbox" javascript error removed, if "AGB Checkbox" is disabled during checkout

    • Confirm response now contains XML-result

    • ServiceUrl has to be set within the plugin settings as required parameter

    • New paymenttypes Skrill Digital Wallet, Skrill Direct, mpass, eKonto, Instant Bank

  • Support of pendingUrl, where orders with pending payments are marked as "delayed"

    • for pending payment methods (eKonto, InstantBank, …​) during state pending a Shopware specific uniqid is used instead of the orderNumber, if you want to get the orderNumber within this pending state please contact our support teams

  • Additional plugin settings supported:

    • Request parameter: maxRetries

    • Sending additional data regarding consumer data (consumerBilling data and consumerShipping data)

Version 1.0.3

  • Improvements regarding support of risk-management system of Shopware for payment methods.

  • Improvements regarding request parameter oderReference which has been used as data type alphanumeric with special characters and is now used as alphanumeric with a variable length of up to 128 characters.
