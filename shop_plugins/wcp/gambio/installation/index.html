Wirecard Shop Plugin for Gambio GX3

Wirecard Shop Plugin for Gambio GX3

Installation guide for Wirecard Checkout Page

This installation guide will show you the step-by-step installation of the plugin to your installed shop system on your web server. Please test your online shop and the configuration of the plugin on a test system before installing the plugin on your production system.

This installation guide is also applicable for Gambio GX2. Any differences in the installation between GX3 and GX2 are specifically indicated in Italics.

Step 1

Upload the zip file containing the Wirecard Checkout Page plugin to the document root of your Gambio GX3 installation and unzip the file. The zip-archive already contains the directory path allowing the plugin to be stored automatically in the right folder.

If you need to unzip the file on your own computer and then upload the plugin, please maintain the directory structure which already exists in the zip file.

Step 2

Open the Administration of your Gambio GX3 installation with your favorite web browser and navigate to the "Payment Systems" menu.

Step 3

Select the Wirecard plugin and install the plugin.

Step 4

In order to update from a previous plugin version, navigate to https://your-shop-url/gambio_updater/ and follow the instructions. First, choose the desired language for installation.

Then, click "Update configuration" if you selected English for installation.

Continue by clicking "Install updates".

You are informed on the successful installation of the updates.

Step 5

If you previously had activated paymentType ELV, then you need to manually reactivate the payment method SEPA Direct Debit after the plugin update is completed.

Configuring the plugin

Step 1

Open the Administration of your Gambio GX3 installation with your favorite web browser and navigate to the "Payment Systems" menu if it is not already open.

Step 2

Please select the Wirecard plugin.

Step 3

After installation you can configure the following elements by clicking on "Edit":

Alias for orders overview Name of payment method in the orders overview. Only applicable for GX3.

Order status

Order status after successful payment with this module. Only applicable for GX3.

Plugin mode

Allows to switch the plugin between live, demo, test or test3d mode (the latter one only applicable for GX3). Please note that no payment is processed in demo or test mode.

Secret

Your secret you received from Wirecard CEE.

Customer ID

Your customerId you received from Wirecard CEE.

Include shop logo URL

Contains the URL to an image on your web server. This image will be displayed during the payment process in the Wirecard Checkout Page.

Shop ID

Unique identifier for your shop. This parameter has to be enabled by Wirecard CEE for your customerId.

Service URL

Contains the URL of an information page in your online shop displaying possibilities for your consumer of how to get in contact with the merchant of the online shop.

Customer statement prefix

Contains a text which is displayed on the invoice of the financial institution of your consumer.

Display text

Optional text which is displayed together with the data of the order.

Sort order of display

Sort order for displaying the payment methods.

Allowed zones

Allows to specify the allowed country zones for payment methods.

Delete orders if payment fails

If enabled, pending orders will be deleted in the order list if payment fails. Note that this setting applies to all Wirecard payment methods.

Delete orders if payment is canceled

If enabled, pending orders will be deleted in the order list if payment is canceled. Note that this setting applies to all Wirecard payment methods.

Use iframe?

If enabled, Wirecard Checkout Page is displayed in an iframe within your online shop, if disabled, the consumer is redirected to Wirecard Checkout Page.

Automatic device detection

The consumer’s device will be automatically detected to optimize the display of Wirecard Checkout Page.

For a detailed description regarding the parameters used for communicating with Wirecard Checkout Page have a look at:

  • Request parameters

  • Response parameters

Customizating and translating text resources

You may customize and translate text resources. Example files are available within the zip-file at /lang/[language code]/modules/payment/wcp<paymenttype>.php

The definitions of text resources regarding error messages for handling a cancelation or a failure of the payment ("cancelURL" and "failureURL") are included in file /lang/[language code]/modules/payment/wcp.php.

FAQ

Why is the order state not shown as successful although payment was successful?

Set the final order state in the drop-down list to ipayment paid as described in Step 3, Configuring the Plugin.
