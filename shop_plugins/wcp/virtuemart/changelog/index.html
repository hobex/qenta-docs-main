Change log

Change log

for VirtueMart and Wirecard Checkout Page

Version 1.7.6 (2018-07-30)

  • Provide Joomla 3.8.10 compatibility.

  • Provide VirtueMart 3.2.14 compatibility.

  • Update language files.

Version 1.7.5 (2018-02-12)

  • Provide VirtueMart 3.2.12 compatibility.

  • Prevent sending of unnecessary data.

Version 1.7.4 (2017-11-29)

  • Prevent checkout issues when Ajax calls "disabled" in settings.

Version 1.7.3 (2017-11-16)

  • Remove payment method @Quick.

  • Prevent possible issues during payment selection.

Tested versions: Joomla 3.7.5 / VirtueMart 3.2.4 with PHP 5.6

Version 1.7.2 (2017-10-04)

  • Rename payment method SOFORT Banking to Sofort./Online bank transfer.

  • Change "SOFORT Banking" logo to Klarna logo.

Tested versions: Joomla 3.7.5 / VirtueMart 3.2.4 with PHP 5.6

Version 1.7.1 (2017-08-29)

  • Add new parameter consumerDeviceId.

Tested versions: Joomla 3.7.5 / VirtueMart 3.2.4 with PHP 5.6

Version 1.7.0 (2017-08-02)

  • Implement financial institutions.

  • Implement payment method Masterpass.

  • Implement payment methods Invoice:RatePAY and Invoice by Wirecard.

  • Implement payment method Installment:RatePAY.

  • Update payment methods.

  • Update pre-checks for payment methods Invoice and Installment.

  • Update payment method logos.

  • Redesign frontend checkout.

Tested versions: Joomla 3.6.5 / VirtueMart 3.2.2 with PHP 5.6

Version 1.6.0 (2017-02-13)

  • Implement update site.

  • Update file structure.

Tested versions: Joomla 3.6.5 / VirtueMart 3.0.18 with PHP 5.6

Version 1.5.0 (2016-12-05)

  • Add new parameter for payment method paysafecard.

  • Improve clearing of shopping basket.

Tested versions: Joomla 3.6.2/VirtueMart 3.0.18 with PHP 5.6

Version 1.4.0 (2016-03-16)

  • Logos for payment methods added.

  • New parameters for background color, customer statement, confirm mail, basket data, layout parameter and basket min/max size added and additional data option split up into "forward consumer billing data" and "forward consumer shipping data".

  • Support request form added.

  • Configuration validator added.

  • Improvements regarding the language strings.

  • New payment methods TrustPay, Invoice B2B, RatePAY: Invoice and Installment, Wirecard: Invoice added.

  • Payment method Click2Pay removed.

Tested versions: Joomla 3.4.1/VirtueMart 3.0.9.4, Joomla 3.4.8/VirtueMart 3.0.14, and Joomla 3.6.4/VirtueMart 3.0.14 with PHP 5.5

Version 1.3.2 (2015-12-02)

  • Update of internal plugin name.

Version 1.3.1 (2015-11-16)

  • Improvements regarding the source code to pass the automatic code review: replaced deprecated directory separator DS with DIRECTORY_SEPARATOR, removed JError, replaced deprecated JRequest method calls, standardized license texts.

Version 1.3.0 (2015-04-15)

  • Plugin updated and improved.

  • paymentType EPAY_BG (payment method ePay.bg), TATRAPAY (payment method TatraPay), and VOUCHER (payment method My Voucher) added.

  • Compatibility with Joomla 2.5.28 / VirtueMart 3.0.6.2 and Joomla 3.4.1/VirtueMart 3.0.6.2 added.

  • Payment method renaming from "Mobile Phone Invoicing" to "paybox".

  • Payment method "Maestro SecureCode" deleted from list of supported payment methods.

Version 1.2.0 (2014-04-02)

  • Plugin updated and improved.

  • paymentType SEPA-DD (formerly ELV) and Trustly (formerly INSTANTBANK) added.

  • Support of confirmation re-delivery attempts if web server did not successfully receive the confirmation.

Version 1.1.0 (2013-07-31)

  • Added payment methods Skrill Digital Wallet, Skrill Direct and mpass.
