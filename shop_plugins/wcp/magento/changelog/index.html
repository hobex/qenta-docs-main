Change log

Change log

for Magento 1.x Wirecard Checkout Page

Version 4.2

Version 4.2.7 (2018-04-10)

  • Update when Ratepay script is loaded.

Version 4.2.6 (2017-12-07)

  • Remove payment method @Quick.

  • Test 3D config update.

Version 4.2.5 (2017-11-29)

  • Prevent possible issue during checkout with free shipping or no shipping.

Version 4.2.4 (2017-09-21)

  • Rename payment method SOFORT Banking to Sofort./Online bank transfer.

  • Change "SOFORT Banking" logo to Klarna logo.

Version 4.2.3 (2017-09-20)

  • Prevent possible issues during update.

  • Add Magento 1.9.3.4 support.

Version 4.2.2 (2017-09-19)

  • Add PHP 7.1 compatibility.

Version 4.2.1 (2017-08-28)

  • Add new parameter consumerDeviceId.

Version 4.2.0 (2017-06-27)

  • Add payment method Masterpass.

  • Remove payment methods mpass and Skrill Direct.

  • Update payment method logos.

  • Add dropdowns for financial institution for payment methods eps-Überweisung and iDEAL.

  • Remove Wirecard as provider for payment method Installment.

Version 4.1

Version 4.1.2 (2017-01-23)

  • Add rollback to previous client library.

Tested version: Magento Community Edition 1.9.3.1 with PHP 5.6

Version 4.1.1 (2017-01-16)

  • Disable all payment methods after installation.

Tested version: Magento Community Edition 1.9.3.0 with PHP 5.6

Version 4.1.0 (2016-12-13)

  • Add new parameter for payment method paysafecard.

  • Use new basket parameters.

Tested version: Magento Community Edition 1.9.3.0 with PHP 5.6

Version 4.0

Version 4.0.4 (2016-04-04)

  • Update of payment method Invoice by payolution.

Version 4.0.3 (2016-03-23)

  • Update of plugin description.

Version 4.0.2 (2016-03-02)

  • Improvements regarding the shopping basket amount.

Version 4.0.1 (2016-02-23)

  • Possibility to edit the front-end labels for payment methods added.

  • Improvements regarding the paths for Magento compiler.

Version 4.0.0 (2016-01-25)

  • Page to send shop configuration to support added.

  • New payment methods My Voucher, TrustPay, Invoice B2B, Installment RatePay / Wirecard, Invoice RatePay / Wirecard added.

  • Update of hash algorithm to HMAC-SHA-512.

Version 3.6

Version 3.6.7 (2015-10-20)

  • Payment methods Credit Card and Maestro SecureCode merged.

Version 3.6.6 (2015-10-16)

  • Improvements regarding storageId with enabled SAQ A compliance.

Version 3.6.5

  • Improvements regarding the payment method display of Invoice for B2B consumers.

Version 3.6.4

  • Improvements regarding request parameter companyVatId which is no longer required for payolution and invoice for B2B consumers. The only required parameter is companyName.

Version 3.6.3

  • Input field CompanyTradeRegistryNumber for paymentType invoice for B2B consumers added.

  • Possibility for merchant to add CompanyTradeRegistryNumber also per customer manually in admin area (Customers→Manage customers→[choose customer to edit]→Account Information).

Version 3.6.2

  • Improvements regarding paymentType invoice for B2B consumers.

Version 3.6.1

  • New feature "PCI DSS SAQ A Compliance" for Wirecard Checkout Seamless added.

  • New fields “SAQ A compliance”, "Show Cardholder Field" and "Show CVC Field" added for Wirecard Checkout Seamless and "PCI DSS SAQ A Compliance" feature.

  • Shopping basket remains visible even if the payment process was canceled.

Version 3.5

Version 3.5.1

  • Use of product specific Controller Action for WCP and WCS confirmUrl.

Version 3.5.0

  • Added support for layout identification

  • Added Magento 1.9 compatibility

  • Added new payment methods ePay.bg and TatraPay

  • Changed parameter paymentType to CCARD in case of Maestro SecureCode payments (MAESTRO is deprecated since 3.30.0)

  • Improved Invoice/Installment when sending additional consumer information

  • Improved order status check in case of browser-history-back for non-English installations

Version 3.4

Version 3.4.0

  • Added compatibility with Magento Compiler

  • Renaming InstantBank to Trustly

  • Added SEPA Direct Debit (ELV still exists for non-SEPA WCS merchants)

  • Changed label of paybox from mobile invoicing to paybox

Version 3.3

Version 3.3.5

  • Support for Magento 1.8 added

  • SEPA return parameters (mandateId, mandateSignatureDate, creditorId and dueDate) for Wirecard Checkout Page and Wirecard Checkout Seamless added

  • IBAN / BIC configuration is default for SEPA Direct Debit within Wirecard Checkout Seamless

Version 3.3.4

  • Consumer notification removed for payment state PENDING.

Version 3.3.3

  • Fixed date of birth for Wirecard Checkout Page integration regarding payment methods INVOICE and INSTALLMENT.

Version 3.3.2

  • Fixed wrong billing-/-shipping state for US/CA consumers.

Version 3.3.1

  • Fixed Multi-currency bug within Wirecard Checkout Seamless interface.

  • Fixed restock after payment cancelation bug.

Version 3.3.0

  • Product renaming Wirecard Checkout Page and Wirecard Checkout Seamless.

  • Added payment methods Skrill Wallet, Skrill Direct, mpass, InstantBank and eKonto.

  • Added Invoive B2B support via payolution.

Version 3.2

Version 3.2.1

  • Improvements regarding consumer birth date within payment methods invoice and installment.

  • Improved wording in German text when consumer is forwarded within payment process in iframe.
