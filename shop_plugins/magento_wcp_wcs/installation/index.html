Wirecard Shop Plugin for Magento Commerce and Magento Enterprise

Installation guide

for Wirecard Checkout Page and Wirecard Checkout Seamless

This installation guide shows the step-by-step installation of the plugin to your installed shop system on your web server. Please test your online shop and the configuration of the plugin on a test system before installing the plugin on your production system.

Step 1

Upload the zip file containing the Wirecard Checkout Page plugin to the document root of your Magento installation and unzip the file. After this, navigate as Admin to "System" → "Configuration → "Payment Methods" to set the corresponding parameters:

Step 2

Enable the payment method by setting the "Enabled" dropdown to "Yes":

Step 3

Please set your CUSTOMERID, SECRET and SERVICEURL:

Step 4 a

For certain payment methods, the seamless integration (Wirecard Checkout Seamless) is available, i.e. the payment data is entered directly on the checkout page of the shopping system and not on the payment provider´s page.

Set to "Enable" for the respective payment method if you use Wirecard Checkout Seamless. If you use Wirecard Checkout Page set to "Disable" (default value):

Step 4 b

As of plugin version v3.6.1, the shop plugin has been updated to fulfill the requirements for credit card payments set forth in the PCI DSS v3.0 which is mandatory for all e-commerce merchants as of 1st January, 2015. Following this standard, merchants web site(s) is not allowed to itself handle sensitive credit card specific data.

Please visit PCI DSS SAQ A Compliance for Wirecard Checkout Seamless for more information and details.

In other words, as a merchant or integrator you may decide whether you integrate Wirecard Checkout Seamless, which requires compliance with the rather stringent SAQ A-EP, or the “PCI DSS SAQ A Compliance” feature which was developed especially for Wirecard Checkout Seamless and ensures compliance with PCI DSS v3.0 and is based on the less stringent SAQ A.

To this effect, some new parameters need to be set in the administration section of the plugin.

"SAQ A compliance" is set to "NO" by default, which means that SAQ A-EP is applicable for your online shop and Wirecard Checkout Seamless will be integrated without the "PCI DSS SAQ A Compliance" feature.

Please note that for using the "PCI DSS SAQ A Compliance" feature for Wirecard Checkout Seamless, "SAQ A compliance" must explicitly be enabled and set to "Yes".

To customize the iframe input fields when using the "PCI DSS SAQ A Compliance" feature, enter a full URL to a CSS file on your server into the "Iframe CSS" field. We also invite you to have a look at the CSS file example which is contained in the plugin: skin/frontend/rwd/default/css/phoenix/wirecard_checkout_page/seamlessiframe.css

Please note that due to security reasons neither url() functions nor @import directives (e.g. external fonts) are allowed in the CSS file. If they are used they will be ignored.

Set the dropdown for Show Cardholder Field, Show CVC Field, Show Issuedate Fields and Show Issuenumber Field to "Enable" if you wish to display these input fields for Wirecard Checkout Seamless and "PCI DSS SAQ A Compliance".

General plugin configuration

To configure the plugin, log in as administrator ("Admin") to your Magento Shop and go to "System" → "Configuration" → "Payment Methods".

Demo mode

As default configuration the plugin is configured for using Wirecard Checkout Page.

If you want to use the plugin in demo mode for Wirecard Checkout Seamless you additionally need to set the optional request parameter shopId to the value qmore.

Production mode

Make sure all of your tests were properly executed before you switch to production mode.

After successful testing you can replace the configuration of the demo mode with your specific values you obtained from Wirecard CEE. Therefore, you only have to change the customerId and the secret which you have received from Wirecard. If required, you also need to adjust the value of the request parameter shopId correspondingly.

Never send the secret by e-mail or instant messaging because it is used for securing the communication between your online shop and our Wirecard servers.

Customizing the GUI of the payment page

You may customize some items of the payment page within the administration section of the plugin:

IFrame If enabled, the Wirecard Checkout Page is displayed within an iframe in your online shop. If disabled, the Wirecard Checkout Page is displayed in the current web browser window during the checkout process. After your consumer finished the checkout process a redirection to your online shop is performed automatically. If you decide to enable iframes be aware that XHTML strict mode does not support iframes.

Seamless

Enable seamless if you want to use Wirecard Checkout Seamless instead of Wirecard Checkout Page.

DISPLAYTEXT

Optional text which is displayed together with the order data.

IMAGEURL

Optional URL to an image which is used as your online shop logo on the Wirecard Checkout pages.

Background color

Background color of the Wirecard Checkout Page as hexadecimal RGB-values like CCDDAA.

Additional customizations

SERVICEURL The URL for a contact data page which resides on your web server. Use a relative URL instead of an absolute URL for this parameter.

Send additional consumer information

You can enable the sending of additional information like shipping and billing addresses.

Enabling input field "Date of Birth"

To use the payment methods "invoice" or "installment" you must enable the input field "date of birth" for your consumers in your Magento configuration. For this, please proceed as follows: go to "System" → "Configuration" → "Customer Configuration" → "Name and Address Options" and set "Show Date of Birth" from the "Optional" to "Required".

"Invoice" or "installment" are only visible if the criteria defined in your contract concluded with the relevant financial Institution (e.g. payolution or RatePAY) are met. For more information please check the payment method specific documentation for your invoice/installment provider:

  • payolution: Invoice and Installment

  • RatePAY: Direct Debit, Invoice and Installment

Specific configurations for Wirecard Checkout Page

Responsive design

Starting with version 1.9, Magento introduced a default responsive design which enables the use of Magento shops on desktops, tablets and smartphones out-of-the-box without any further adaptations regarding mobile devices.

If you use Wirecard Checkout Page for the checkout process in your shop, you have the possibility to display a device-specific optimized checkout page by using the feature "Detect Layout".

If this feature is enabled by setting the value to "Yes", an automatic consumer device identification (either desktop, smartphone or tablet) is done within the plugin. If a mobile device is identified, the corresponding request parameter layout is automatically set to the value tablet or smartphone and an optimized presentation of Wirecard Checkout Page for mobile devices is displayed in your shop.

To improve usability for smartphones and tablets, the plugin automatically disables the embedding of the checkout page within an iframe on tablets and smartphones and redirects the consumer to the checkout page of Wirecard. After the checkout process is completed by the consumers, they are redirected to the shop.

If you use an earlier version of Magento (Magento v1.8) and you have implemented a responsive or mobile design on your own, this "Detect Layout" feature may also be used.

Please contact our sales teams if you wish to use the feature "mobile layouts for WCP" in your Magento shop.

Specific configurations for Wirecard Checkout Seamless

Wirecard Checkout Seamless within an iframe

If you configure the plugin for using Wirecard Checkout Seamless in combination with an iframe, this iframe will "overlay" the checkout page. Therefore it is required to have a 3rd party library named prototypeJS at least in version 1.7 or later.

This library is already included in Magento 1.6.0 or later. If prototypeJS 1.7 or later is not available within your online shop, Wirecard Checkout Seamless will redirect to a new HTML page containing an iframe.

FAQ

Is the plugin compatible with the Magento Compiler?

Yes, compatibility is given starting with plugin version 3.4.0. To check the "Compiler Status" in your Magento Administration Panel go to "System" → "Tools" → "Compilation". The "Compiler Status" should be enabled.

I performed an update following the steps described in "Updating from versions earlier than V3.3.0 to the current version", however, the former payment methods are still displayed. How can I solve this problem?

Deleting the config of the previous plugin in the database has not succeeded. You need to delete the config manually by executing the following SQL statement “DELETE FROM core_config_data WHERE path LIKE '%qenta%';” and clear the configuration cache again.

How to update from versions earlier than V3.3.0 to the current version?

When you are updating from a plugin version earlier than V3.3.0 proceed as follows:

  1. Perform a backup of your online shop and the database.

  2. Clear the configuration cache.

  3. Remove the following files and folders only:

    • /app/design/frontend/base/default/layout/qpay.xml

    • /app/design/frontend/base/default/template/qpay

    • /app/design/adminhtml/default/default/template/qpay

    • /app/code/community/Phoenix/WirecardQPay/controllers

    • /app/code/community/Phoenix/WirecardQPay/etc/system.xml

Why does the ERP system assume an order although the payment has not been successfully completed yet?

For some payment methods the payment state may be set for a few moments to "Pending" before being set to "Pending Payment". In some cases, the ERP system might as a result assume an order although the payment has not been successfully completed yet.

In order to solve this, you need to enter the following command in the config.xml () of the respective payment method (Sofortüberweisung is used here by way of example):

<wirecard_checkout_page_sofortueberweisung>
…​
<order_status>holded</order_status>
…​
</wirecard_checkout_page_sofortueberweisung>

When does the input field "Company Trade Registration Number" appear?

The field to enter the "Company Trade Registration Number", which applies to B2B consumers, only appears if the consumer to your online shop has also filled in the field "Company Name".

FAQ for Wirecard Checkout Seamless

Your consumer who uses gets an error message regarding a missing CVC after entering the credit card number, even if the option for the corresponding check is not activated?

The CVC has to be used for all types of cards where a CVC is required (like Visa or MasterCard). If you activate that check in the back-end of your shop, only cards without CVC can be used by your consumers (like Maestro or UATP).
