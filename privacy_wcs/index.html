Information on the protection of personal data for QMORE Checkout Seamless

Information on the protection of personal data for QMORE Checkout Seamless

Overview

This chapter provides information on all data that are processed by our solution QMORE Checkout Seamless.

When ordering QMORE Checkout Seamless, you may acquire

  • 1 basic solution, optionally extendable with

  • 1 to N payment methods, and/or

  • 0 to N optional and additional features

Go to section “Payment method selection” in chapter Basic Solution to simulate the processing of data (parameters) for a given payment transaction and the selected payment method. Furthermore, section Additional Features provides additional information on the processing of data (parameters) for each feature that was added to the basic solution.

Should you have any further questions regarding data protection details and individual parameters, e.g. data subjects, categories of data, etc., consult privacy.adoc#DataProtectionInformationRegardingIndividualParameters for additional information.

Basic Solution

Data processing according to payment method

Go to section “Payment method selection” to simulate the processing of data (parameters) for a given payment transaction and the selected payment method. The displayed data refer to

  • data, which the merchant or consumer sends to the QENTA payment interface (so-called input data),

  • data, which QENTA returns to the merchant or consumer (so-called output data).

All indicated parameters need to be processed to ensure that the solution QMORE Checkout Seamless, basic version, meets our contractual obligations to the merchant (purpose of the processing of data):

The displayed parameters are subdivided into

  • parameters processed irrespectively of the payment method (standard)

  • parameters processed dependent on the respective payment method (payment method)

  • parameters entered by the consumer (consumer interaction)
