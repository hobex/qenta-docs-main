PayPal

PayPal

General information

To integrate and offer the payment type PAYPAL for your consumers in your web shop you need to fulfill some important requirements:

  • All required data of your consumer are available.

  • Submit additional parameters which are required by PayPal for processing the payment of your consumer.

  • Your consumer may change the shipping address within the payment in PayPal. Therefore we return the shipping address together with other payment details within the response parameter to your web shop for further processing within your fulfillment process.

  • Setup third party permissions for Wirecard CEE to your PayPal account as described below.

If these requirements are not fulfilled, please do not offer this payment type to your consumer within your web shop.

You can find more details regarding PayPal at https://www.paypal.com.

If your web shop is enabled for payment type PAYPAL it is required that you use additional request and response parameters when integrating the QPAY Checkout Page to your shop. Especially these request parameters have to been set whenever you (or your consumer) set the payment type to PAYPAL or to SELECT.

If the payment has been accepted by PayPal you will get as response the parameter paymentState with the value ”SUCCESS”.

We recommend to store the transaction id of the payment as returned by the parameter gatewayReferenceNumber within the data regarding the corresponding session and order for documentation purposes.

At a later time you have also the possibility to get the transaction id via the Wirecard Payment Center by searching for the specific payment and then using the details view for the “Gateway-Referencenumber”.

Setup of 3rd party permissions to your PayPal account

To enable the integration of PayPal to your web shop within the QPAY Checkout Page we need to get some "third party permissions" to your PayPal Business Account for accessing PayPal related payment information.

This can be done within the following steps:

  1. Please register at https://www.paypal.com for a PayPal Business Account.

  2. Log in with this account at PayPal.

  3. Navigate to "My Account" → "Profile" and select "API Access".

  4. Select "Grant API permission".

  5. Enter "office_api1.qenta.at" as the "Third Party Permission Username" and click on "Lookup".

  6. Select from the list of available permissions the following items:

    • "Use Express Checkout to process payments."

    • "Use Express Checkout to process mobile payments."

    • "Obtain information about a single transcation."

    • "Authorize and capture your PayPal transactions."

    • "Issue a refund for a specific transaction."

    • "Charge an existing customer based on a prior transaction."

    • "Create and manage Recurring Payments."

  7. Finally, please send us your PayPal registration e-mail address to our Support teams.

FAQ

Which language is presented to your consumer within the PayPal pages?

If the parameter “consumerShippingCountry” is set PayPal will display its own pages in the language of that country. Otherwise if this parameter is not set PayPal will use the language configured within the web browser of your consumer.
